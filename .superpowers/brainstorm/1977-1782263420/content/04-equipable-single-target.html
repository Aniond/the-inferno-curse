Single Target Abilities — Equipable Model

Key insight: Abilities are equipable (learned vs equipped). Starting with Single Target templates.

Core Principle (from you)

Abilities must be equipped to use. They are not automatically available just because learned via absorption. Equipping has consequences (corruption, slot limits, build choices).

Usable Template: SingleTargetAbility

Example: strike.tres (or runtime dict)
id = "strike"
display_name = "Strike"
description = "Basic physical attack."

type = "single_target"

# Targeting

# CT Interaction (very important)

# Costs

Why equipable makes sense here

  • Matches absorption fantasy: you "take" the power but have to choose to carry it.
  • Creates meaningful decisions (which 4 abilities do I run with?).
  • Corruption feels real — it's the cost of what you choose to equip.
  • Future proofs for Buff/Curse and Traits (some might be passive when equipped).

Open questions this raises

  • How many equip slots? (fixed 4? scales with level?)
  • Do equipped abilities give passive bonuses even when not used?
  • Can you swap equipped abilities outside combat only?
  • Should Single Target heals have different equip cost than attacks?
