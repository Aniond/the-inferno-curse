Orb Tree Rules Update: Equip Limit + Shadow/Light Merging Paths

You can only equip ONE instance of each unique orb, even if you have multiples stacked in the tree. Tree has branching Shadow (Sinner) and Light (Saint) paths that can merge at key nodes for hybrid options.

Equip Limit (per your reminder)

For each unique orb (e.g. "Shadow Claw"):

  • You can place as many copies as you want in the tree (no hard cap) to stack power and sanity cost.
  • You can only ever have ONE copy equipped for active use at a time.
  • The power from all stacked copies in the tree applies to the single equipped version.

This keeps the "slot them both" idea (tree for passive growth + equip for active use) while preventing unlimited active copies of the same orb.

Shadow / Light Branching Paths with Merges

Tree Structure Concept

Start Node (neutral base orbs)
↓
Branch A: Shadow Path (Sinner) — raw power, higher base sanity/corruption, aggressive effects
    ↓
    Merge Points (hybrid nodes)
    ↑
Branch B: Light Path (Saint) — purified, lower corruption, utility/healing focus, Church synergy

Merge Examples
• A Shadow Damage orb can merge with Light nodes to gain holy damage vs Limbo creatures while keeping some raw power.
• A Light Heal orb can merge with Shadow nodes for "corrupted healing" that also damages enemies or drains sanity from foes.
• At merge nodes you choose the blend — but high merges may have unique risks/rewards (e.g. temporary sanity spikes or world corruption surges).

This replaces the strict "Dark and Light cannot coexist" with controlled, node-based merging for more build variety.

Impact on Single Target Template

Each Single Target orb (Damage or Heal) now has:

  • Its flat base_sanity_cost and base_corruption_value (as previously set).
  • Tree stacking applies to the unique orb type regardless of path.
  • When equipped, only one version is active, but it benefits from all tree investments on that orb (including any merged shadow/light nodes).
  • Path choice happens at the tree nodes you invest in for that orb.

Benefits

  • Equip limit keeps active loadout clean and meaningful (you can't spam 5 Shadow Claws at once).
  • Merging paths give real choice and hybrid builds without breaking the Saint/Sinner fantasy entirely.
  • Fits the "you can only equip one per the skill tree" while still allowing deep investment via stacking.
  • Merges can create interesting orbs that bridge dark/light (e.g. "Shadow Strike" that occasionally heals on crit if merged with light nodes).
  • Church purification can "reset" an orb to a pure Light version, clearing some shadow investments.

Tradeoffs / Things to Balance

  • Merges might dilute the purity of Saint vs Sinner if overused — need strong narrative or mechanical cost for deep merges.
  • Need to decide if merges are permanent per orb copy or can be respecced.
