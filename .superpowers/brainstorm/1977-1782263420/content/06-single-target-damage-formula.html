Single Target Damage — Base Damage Modified by Stats

Equipable template with explicit damage calculation

Your clarification

The ability carries a base_damage that gets modified by the user's stats at the time of use.

Updated Damage Variant Template

strike.tres (Damage variant)
id = "strike"
display_name = "Strike"
type = "single_target"
variant = "damage"

base_damage = 8
scaling_stat = "STR"
stat_multiplier = 1.0 # damage added per point of stat

# Targeting
range = 1
is_ranged = false

# CT
ct_target_delay = 12
ct_self_gain = 0
action_cost = 1

mp_cost = 0
corruption_on_equip = 1
corruption_on_use = 0

theme_tags = ["martial"]

Calculation (usable formula)

Damage Resolution
final_damage = base_damage + (user.scaling_stat_value * stat_multiplier)

Then apply the normal combat modifiers:
+ height bonus
+ flank / pincer
- target defense + cover
* weather damage_mult (if any)

Example: User has STR 7, Strike (base 8, mult 1.0)
final_damage = 8 + (7 × 1.0) = 15
(before height/flank/cover/weather)

Why this shape is usable

  • base_damage gives the "identity" of the ability (weak vs strong)
  • stat_multiplier lets strong abilities scale harder with high stats
  • scaling_stat can be different per ability (STR for physical, INT for spells)
  • Works the same whether the user is player or (future) NPC
  • Easy to author on monster sheets for absorption

Pros

  • Simple, readable template
  • Base + stat mod is intuitive
  • Can tune per ability without changing core formula
  • Still feeds into existing combat_state.calculate_damage

Open decisions

  • Should stat_multiplier default to 1.0 or be ability-specific only?
  • Do we want a minimum damage floor? (e.g. always at least base_damage)
  • How does this interact with natural_power on monsters?
