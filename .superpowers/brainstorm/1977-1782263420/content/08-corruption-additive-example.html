Single Target Corruption — Additive Increment (Your Exact Example)

Confirmed: each ability's corruption value increases by the base cost of every newly equipped ability

Your Rule (Additive)

Skill A starts with 1% corruption cost.
You equip a second skill that has a 1% base cost → Skill A's corruption value increases by that 1% amount.

This applies symmetrically and scales with higher base costs and more abilities.

Concrete Walkthrough (your example)

Step by step
Only Skill A equipped
A base=1% → effective corruption for A = 1%

Add Skill B (base=1%)
A base=1% + B's 1% = 2%
B base=1% + A's 1% = 2%

Add Skill C (base=3% higher skill)
A = 1% +1% +3% = 5%
B = 1% +1% +3% = 5%
C = 3% +1% +1% = 5%

How the increment works in the template

effective_corruption = base_corruption + sum(base_corruption of all other equipped abilities)

This means:

  • More equipped abilities = every ability's value goes up
  • Higher base cost skills increase everyone else's corruption more
  • The value is stored/calculated per ability, not just a global total
  • Total player corruption pressure = sum of all effective_corruption

Strengths of this additive model

  • Very clear and intuitive ("each new skill increases the cost of the ones you already have")
  • Strongly punishes loading up on many abilities
  • Higher skills (higher base) naturally punish the whole loadout more
  • Fits the "price of power" and Limbo/Florence flavor perfectly

Remaining decisions

  • Is the increase applied to every equipped ability (including the new one), or only to previously equipped?
  • Does "higher skills" also mean abilities can level up and raise their own base_corruption over time?
  • Is the % a direct corruption % or a multiplier on some other system?
