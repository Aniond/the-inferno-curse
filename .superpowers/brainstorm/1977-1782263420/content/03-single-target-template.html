Single Target Skills — Usable Template Design

Starting here as requested. Focus: Attack + Heal variants for absorption (Normal tier first)

Design Goal

Create usable, extensible templates that monsters can offer via absorption, that plug into the current tactical combat (grid, CT action system, height, status, weather mods).

Proposed Single-Target Ability Template (Data Shape)

AbilitySingleTarget.tres (example Resource)
id: "strike"
display_name: "Strike"
description: "A basic melee attack."
type: "single_target"
category: "attack" # or "heal"

# Targeting
range: 1
is_ranged: false
requires_los: true

# Effect
base_damage: 8
base_heal: 0
power_stat: "STR" # or INT/FTH for spells

# CT / Action interaction
ct_self_bonus: 0 # + to own current_ct (speed up turn)
ct_target_penalty: 15 # - to target's current_ct
action_cost: 1 # how much this action advances everyone

# Status / Mods (optional)
apply_status: ""
status_duration: 0

# Cost & Corruption
mp_cost: 0
corruption_per_use: 0

# Theme tags for jobs
theme_tags: ["martial"]

Two Core Variants

Attack Variant (Normal)
Strike / Dagger Cut / Gnaw
- Damage based on power_stat
- CT penalty on target (delays their turn)
- Small chance to apply minor status
- Low or zero MP
- Low corruption
Heal Variant (Normal)
Guarigione / Minor Heal / Bandage
- Positive HP on single ally (or self)
- Can give small CT bonus to target
- Faith or Intellect scaling
- Low MP
- Very low corruption (or none)

How it works in current combat flow

  • During ACT phase → player chooses skill instead of basic attack
  • Grid targeting reuses existing get_attackable_targets (single cell)
  • On use: resolve damage/heal + direct current_ct adjustments
  • Call combat_state.advance_ct( action_cost )
  • If status, use the existing apply_status_effect system
  • Weather mods already apply (damage_mult, etc.)

Pros of starting with this template

  • Reuses almost all existing combat code
  • Easy to hook into monster normal_ability
  • Immediately testable in battle_test_map
  • Covers both "att" and "heal" in one pattern

Tradeoffs / Things to decide

  • Flat numbers vs formulas (power * multiplier + flat)
  • Should heals also cost an action / advance CT?
  • Corruption only on attack variants or on powerful ones?
  • Will these be Resources or still just strings + data dict for now?
